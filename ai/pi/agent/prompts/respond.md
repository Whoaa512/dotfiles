Investigate and draft a response for: $@

## Process

### 1. Understand the context
- Read the URL/thread/doc using appropriate tool (slack-cli, gdoc, fetch, etc.)
- Identify: who's asking, what they want, what the tone is

### 2. Cross-reference source code
- If the topic relates to code we own, read the relevant source
- Verify claims/assumptions against actual implementation
- Note any inaccuracies in the thread

### 3. Draft response
- Write in user's voice (use cj-voice skill)
- Format for the target platform:
  - **Slack:** backticks for code, short paragraphs, emoji sparingly
  - **PR comment:** reference specific files/lines
  - **Google Doc:** full sentences, section headers if long
  - **Email:** professional but casual
- Be pithy — say more with less
- Lead with the answer, context after

### 4. Output
Present the raw markdown/text (not rendered) so it can be copy-pasted directly.

Copy to clipboard when done.

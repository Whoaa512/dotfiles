# User Guide

Describe what you want. Complex work happens behind the scenes. Results arrive.

## What Happens

```
Your Request → Understanding → Parallel Work → Synthesis → Results
```

You don't need to know the details. Just describe the goal.

---

## Tips for Best Results

### Be Specific

| Instead of... | Try... |
|---------------|--------|
| "Review the code" | "Review PR #123 focusing on security" |
| "Fix the bug" | "Fix the login bug after password reset" |
| "Make it faster" | "Optimize database queries in user service" |
| "Add a feature" | "Add dark mode with a toggle in the header" |

### Provide Context

```
"Add authentication — we're using Next.js with Prisma"
"Refactor to async/await — this is a Node.js backend"
"Review for security — this handles payment data"
```

---

## FAQ

**Can I see more detail?**
Ask: "Show me more detail" or "Walk me through your process."

**Can I do things step-by-step?**
Say: "Let's do this step by step" or "Check with me before each part."

**Can I interrupt?**
Yes. Just send a new message.

**Why did I get asked a question?**
Questions appear when:
- Request could be interpreted multiple ways
- There are meaningful choices
- Action is destructive or hard to reverse

---

## Getting Better Results

1. **Be more specific** — add context about codebase, goals, constraints
2. **Break it down** — large vague requests → focused ones
3. **Provide examples** — "like how X works in Y file"
4. **Ask why** — "why did you do it that way?"
